# Emberfall Game Design Document Snapshot

## Fantasy

The player is a ruin runner crossing a fallen frontier where traversal mastery, environmental reading, and compact combat skill decide survival.

## Core Loop

1. Enter a hostile zone
2. Read terrain and environmental clues
3. Overcome traversal or combat obstacles
4. Earn progression currency or unlock fragments
5. Return to earlier routes with expanded mobility

## Pillars

- Traversal must feel readable before it feels difficult
- Combat depth should come from timing, spacing, and encounter composition
- The world should communicate narrative context through spaces before exposition

## First Slice Targets

- One hub-adjacent ruin path
- One mini-boss encounter
- One traversal unlock that changes route interpretation
- One optional side route that rewards observation

## Explicit Non-Decisions

This document does not define:
- Unity component breakdown
- Unreal gameplay framework classes
- Godot node composition
- engine-specific save/load architecture
