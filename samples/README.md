# Samples

Complete, playable games generated entirely by the `/full-game` command from this scaffold. No manual coding involved — each game was produced end-to-end from a single natural-language prompt and kept as verbatim harness output.

Conventions for adding and maintaining samples: see [CONVENTIONS.md](CONVENTIONS.md).
Cross-harness analysis of the two controlled experiments: see [harness-comparison.md](harness-comparison.md).

---

## PirateInvaders

**Genre:** 2D Arcade Shooter (Space Invaders variant)
**Platform:** Web (HTML5 — Phaser 3)
**Harness:** Codex (April 2026)
**Features:** Pirate-themed waves, power-ups, online leaderboard

```
/full-game a pirate-themed Space Invaders game for HTML
```

**Run:** `cd samples/PirateInvaders && npm install && npm run dev`

## PirateInvadersImagesGen

**Genre:** 2D Arcade Shooter (Space Invaders variant)
**Platform:** Web (HTML5 — Phaser 3)
**Harness:** Codex (April 2026)
**Features:** Same gameplay as `PirateInvaders`, but with gameplay and UI raster assets generated through Codex `$imagegen`

This sample exists as a scaffold integration demo: it keeps the original texture keys and gameplay logic, swaps procedural runtime textures for generated image assets, and shows how to persist accepted outputs inside the workspace (see `generated-assets.json` and `npm run validate:generated-assets`).

**Run:** `cd samples/PirateInvadersImagesGen && npm install && npm run dev`

## Tetris2DMutation

**Genre:** 2D Puzzle / Arcade (Tetris variant)
**Platform:** Web (HTML5 — Vanilla Canvas)
**Harness:** Claude Code (April 2026)
**Features:** Mid-fall piece mutation, 3-slot basket system, procedural audio, touch controls

```
/full-game a 2D Tetris-like game for HTML where sometimes the pieces mutate mid-fall and there
is a basket option where the player can store a piece they don't want to use, and on the next
turn they can use pieces from the basket to replace the one that is falling.
Put it in the Tetris2DMutation folder
```

**Run:** `cd samples/Tetris2DMutation && npx serve .`

## Los Renacidos — three-harness experiment (2026-05-14)

The same Spanish-themed roguelike generated by three harnesses from the same source material. Prompt not recorded verbatim. Full analysis in [harness-comparison.md](harness-comparison.md).

### LosRenacidos_Claude

**Genre:** 2D Action Roguelike · **Platform:** Web (Vanilla Canvas, ES modules)
**Harness:** Claude Code
**Highlights:** Modular state-machine architecture, 3-phase combat with parry, node-graph run map, 2-phase boss, quest + NPC dialogues, versioned saves

**Run:** `cd samples/LosRenacidos_Claude && npx serve .` (ES modules — needs a server)

### LosRenacidos_Codex

**Genre:** 2D Action Roguelike · **Platform:** Web (Vanilla Canvas, single file)
**Harness:** Codex
**Highlights:** Only variant with generated bitmap art (~11.5MB PNGs with procedural fallback), corruption difficulty system, settings + pause, runs without a server

**Run:** open `samples/LosRenacidos_Codex/index.html` directly, or `npx serve .`

### LosRenacidos_OpenCode

**Genre:** 2D Action Roguelike · **Platform:** Web (Vanilla Canvas, ES modules)
**Harness:** OpenCode
**Highlights:** Most ambitious architecture (8 scenes, 8 systems, 2 biomes, context music loops)
**Known issue (kept as harness-output evidence):** does not boot as shipped — `src/scenes/EventScene.js` default-imports `DialogueUI`, which only has a named export, so the module graph fails to load. Several systems (`CombatSystem`, `CorruptionSystem`) are dead code.

**Run:** `cd samples/LosRenacidos_OpenCode && npx serve .` — expect a black screen until the import bug is considered (do not fix in place; regenerate per CONVENTIONS.md)

## pacmanAI — three-harness experiment (2026-05-14)

The same Pac-Man-style game with AI ghosts generated by three harnesses. The original Spanish prompt is preserved inside `pacmanAI_Claude/README.md`. Full analysis in [harness-comparison.md](harness-comparison.md).

### pacmanAI

**Genre:** Maze Arcade · **Platform:** Web (Vanilla Canvas, ES modules)
**Harness:** OpenCode
**Highlights:** Most complete arcade feel — scatter/chase wave table, level scaling, procedural music loops, top-10 high-score table

**Run:** `cd samples/pacmanAI && npx serve .`

### pacmanAI_Claude

**Genre:** Maze Arcade · **Platform:** Web (Vanilla Canvas, ES modules)
**Harness:** Claude Code
**Highlights:** Best-engineered variant — DI context, snapshot pause/resume, 6-mode ghost state machine with house/leaving, swipe + on-screen D-pad mobile controls

**Run:** `cd samples/pacmanAI_Claude && npx serve .`

### pacmanAI_Codex

**Genre:** Maze Arcade ("Neon Maze Chase") · **Platform:** Web (Vanilla Canvas, single file)
**Harness:** Codex
**Highlights:** Smallest footprint (~768 LOC), settings screen with persisted volume/mute, ARIA-labeled touch buttons, runs without a server

**Run:** open `samples/pacmanAI_Codex/index.html` directly, or `npx serve .`
