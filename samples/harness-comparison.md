# Harness Comparison — /full-game Output

Two controlled experiments: the same game generated by three harnesses
(Claude Code, Codex, OpenCode) from approximately the same prompt, on
2026-05-14. Samples are verbatim harness output (see `CONVENTIONS.md`), so
the differences below are differences between harnesses, not between
editors. Code-level findings were verified by reading the sources.

## Experiment 1 — Los Renacidos (Spanish-themed roguelike)

| Dimension | LosRenacidos_Claude | LosRenacidos_Codex | LosRenacidos_OpenCode |
|---|---|---|---|
| Files / JS LOC | 31 / ~2,800 | 14 / 1,326 (one file) + 11.5MB PNGs | 36 / ~3,500 |
| Organization | Modular ESM (states/systems/entities/data/world) | Single-file IIFE + generated bitmap assets | Most granular (scenes/systems/entities/ui/utils) |
| Docs | Slice GDD + TDD + excellent README | Concise GDD + TDD + README (translated to English) | TDD only; no README; original 32KB GDD committed twice (byte-identical files) |
| How to run | Server required (`npm run dev`) | Opens directly from `file://` (no ES modules) | Server required — **but fails to boot** |
| Runs as shipped? | Yes | Yes (most frictionless) | **No** — `EventScene.js:4` default-imports `DialogueUI`, which only has a named export; the module graph fails and the game black-screens |
| Combat depth | 3-phase attacks, parry window, dash i-frames, 2-phase boss | Light/heavy, block, dodge, ranged archers, corruption-scaled difficulty | Attack/block/dodge, projectiles, obstacle collision, camera, damage numbers |
| Pause / settings | Neither | Both (volume persisted) | Both |
| Audio | Procedural SFX | Procedural SFX + ambient drone | Procedural SFX + context music loops (only variant with music) |
| Standout flaws | Duplicated wall-collision code (player/enemy), double swing SFX, divergent XP curve constants | God-class, magic numbers everywhere, 11.5MB binaries in the repo | Boot-breaking bug; `CombatSystem`/`CorruptionSystem` are dead code (imported by nothing); room-gen duplicated; quest data typo (`hierarchy_moonstalk` vs `hierba_moonstalk`) makes the gather quest uncompletable |

**Verdict:** Claude shipped the best-engineered and best-documented variant.
Codex traded architecture for reliability and reach (only variant that runs
from `file://`; most complete screen flow). OpenCode attempted the most
ambitious architecture and feature set but shipped it broken: a fatal import
mismatch prevents loading, and several showcase systems are dead code.
Ranked on shipped quality: **Claude > Codex > OpenCode**; ranked on
architectural ambition alone, OpenCode would lead — exactly the gap between
scaffolding and verification.

## Experiment 2 — pacmanAI (Pac-Man-style with ghost AI)

| Dimension | pacmanAI (OpenCode) | pacmanAI_Claude | pacmanAI_Codex |
|---|---|---|---|
| Files / code LOC | 20 / ~1,838 | 20 / ~1,692 | 8 / ~768 |
| Organization | Modular ESM (entities/systems/states) | Modular ESM (identical layout, independently written) | Single-file IIFE + CSS ("Neon Maze Chase") |
| How to run | Server required | Server required | Opens directly from `file://` |
| Ghost AI | 5 modes, greedy tile-choice, legal eyes-return, 4 authentic personalities | 6 modes incl. HOUSE/LEAVING with release delays, 4 personalities; eaten ghosts fly through walls (shortcut) | Scatter/chase + frightened, 4 approximated personalities; eaten = teleport with delay (no eyes mode) |
| Maze | Classic 28×31 numeric grid | Classic 28×31 string grid (most readable) | Custom 23×21 ASCII layout |
| Audio | SFX + LFO-driven music loops, frightened music speed-up | SFX only (ADSR envelopes); `ambient` declared but unused | SFX only + settings screen with volume/mute persisted |
| Extras | Mode-wave table, level scaling, top-10 high-score table with dates | Best mobile controls (swipe + on-screen D-pad), best README (includes the original Spanish prompt) | Runs from file://, ARIA-labeled touch buttons, smallest footprint |
| Standout flaws | Dead tunnel-speed logic, identical if/else branches, pause via FSM-internals mutation, duplicated scatter-target constants | Unused `moveToward`, no-op keyup listener, no music, no per-level scaling | `s` key conflict (WASD + settings), redundant frightened logic, least faithful arcade behavior |

**Verdict:** OpenCode produced the most *complete game* (music, high-score
table, arcade-accurate waves) with the most code smell. Claude produced the
best-*engineered* variant (injected context, snapshot-based pause, richest
ghost state machine, best docs and mobile controls) but skipped music and
difficulty scaling. Codex optimized for pragmatism: a quarter of the code,
zero-friction run, a unique settings screen — and the least faithful Pac-Man.

## Cross-experiment patterns

Consistent across both experiments:

1. **Claude Code**: best architecture and documentation discipline (state
   machines, dependency injection, data-driven content, README quality), at
   the cost of some scope (music, settings/pause in the roguelike) and minor
   dead-code remnants. Most extensible starting point.
2. **Codex**: pragmatic minimalism — single-file builds that run straight
   from `file://`, smallest footprint, uniquely ships settings screens; least
   faithful to the prompt's full ambition and least extensible.
3. **OpenCode**: highest ambition and feature count (music systems, broadest
   scope), weakest verification — shipped the only non-booting game, the most
   dead code, and FSM-internals hacks. Output needs a review pass before trust.

Scaffold lessons fed back (see `/learn`):

- `/full-game` should end with a boot smoke test — "the game loads without
  console errors" — which would have caught the LosRenacidos_OpenCode failure
  at generation time. The web layer's `rules/web/testing.md` now encodes this.
- Recording the exact prompt per sample is mandatory going forward
  (`CONVENTIONS.md`); only pacmanAI_Claude preserved it spontaneously.
