# Sample Conventions

Samples are complete games generated by the `/full-game` command, kept as
verbatim harness output for comparison. These conventions keep them useful as
comparison data without holding them to scaffold authoring style.

## What a sample is

- The unmodified output of one `/full-game` run on one harness.
- Evidence of what the scaffold produces, and the raw material for comparing
  harnesses against each other (see `harness-comparison.md`).

## Per-sample requirements

Each sample folder entry in `samples/README.md` must record:

- **Harness** that generated it (Claude Code, Codex, OpenCode, ...) and the
  generation date (commit date is acceptable).
- **Prompt** used, verbatim when available; `prompt not recorded` otherwise.
  Record the prompt going forward — comparisons are weak without it.
- **Run instructions** that work from a fresh clone.

## Source-of-truth rule

A sample is **modular + served**, not single-file-bundled:

- ES-module sources stay in `src/`; the game runs via a local server
  (`npx serve .` or the project's `npm run dev`). Opening `index.html` over
  `file://` is not a supported path for module-based samples.
- Never inline `src/` modules into `index.html` to make `file://` work — that
  creates two sources of truth inside the sample. If a single-file build is
  ever wanted, it must replace `src/` entirely, not duplicate it.

## Hygiene

- `node_modules/`, build caches, and local state stay untracked (gitignored).
- Samples are excluded from scaffold markdown lint — they are external
  artifacts, not scaffold source.
- Do not retro-edit sample code: fixing or improving a sample destroys its
  value as harness-output evidence. Regenerate instead, as a new sample.

## Feedback loop

After adding a sample, capture what it demonstrates:

- Update `samples/README.md` (entry) and `harness-comparison.md` when the
  sample is part of a multi-harness experiment.
- Feed durable lessons back into the scaffold via `/learn` and `/evolve`
  (rules, skills, or command refinements — not sample edits).
