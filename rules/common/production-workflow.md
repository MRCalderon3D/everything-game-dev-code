# Production Workflow

## Purpose
Define the shared planning, milestone, estimation, dependency, and execution model for game development work.

## Scope
Applies to concept, pre-production, production, beta, release, and post-release phases.

## Workflow Principles
- Plan around validated outcomes, not activity volume.
- Make dependencies visible before they become blockers.
- Use milestones to de-risk, not merely to rename calendar dates.
- Keep scope, quality, and schedule trade-offs explicit.

## Milestone Model
- Common milestone types should include prototype, vertical slice, first playable, content complete, alpha, beta, release candidate, and launch as applicable.
- Each milestone must define purpose, entry criteria, exit criteria, and owner.
- Milestones should be tied to risk retirement, quality bar validation, or content/state maturity.

## Backlog Rules
- Work items must be scoped, owned, and linked to a milestone or operating goal.
- Large items should be decomposed before they become active execution work.
- Tasks should include dependencies, acceptance criteria, and affected disciplines.
- Scope cuts and descopes must be recorded, not handled informally.

## Estimation Rules
- Estimates should reflect uncertainty, dependencies, and external review needs.
- Novel or risky work should be prototyped before being treated as routine implementation.
- Re-estimate when assumptions materially change.

## Definition of Ready
- Work should not enter implementation without objective, scope, dependencies, acceptance criteria, and source-of-truth references.
- Work missing design or technical ownership is not ready.

## Definition of Done
- A task is done only when implementation, validation, documentation, and integration expectations are satisfied.
- Done must account for QA, telemetry, accessibility, and release implications where relevant.

## Risk Rules
- Maintain a visible risk register with owner, probability, impact, mitigation, and trigger.
- Revisit top risks at milestone reviews.
- Hidden uncertainty is a planning failure.

## Communication Rules
- Status reporting should distinguish progress, blockers, risk changes, and decision needs.
- Escalations should happen as soon as milestone impact becomes plausible.

## Deliverables
- Milestone plan.
- Backlog hygiene rules.
- Risk register.
- Ready and done definitions.
- Status reporting template.

## Done Criteria
Production workflow is healthy when work is planful, owned, reviewable, and resilient to cross-discipline dependencies.
